# Coordination value of interoperable digital platforms for hourly positive energy districts

Reproduction code for the paper *Closing the last 20%: the coordination value of
interoperable digital platforms for hourly positive energy districts*.

The code models a multi-carrier prosumer community that couples electricity and
heat behind a shared grid connection with a net-export hosting limit. The same
physical system is solved as a centralized community problem (a mixed-integer
quadratic program) and as a price-taking peer-to-peer market equilibrium. An
ordered five-stage waterfall (DE0, M1, IND, M2, M3) decomposes the coordination
gap into the value of shared forecasts, perfect information, internal trading,
and grid-aware pricing.

## Repository structure

| File | Contents |
|------|----------|
| `config.py` | Run settings, technology parameters, zonal market calibration, and the six district-community definitions. |
| `data.py` | Calibrated hourly profiles (weather, photovoltaic availability, load, heat) reproducible from a fixed seed. |
| `model.py` | Gurobi centralized planner (`solve_planner`) and the decentralized waterfall (`solve_waterfall`), including the equilibrium price recovery and the illustrative dual-decomposition routine. |
| `experiments.py` | All experiment runs (main matrix, Pareto front, sensitivities, uncertainty, robustness, market power) and the independent CVXPY/Clarabel audit. Caches every result to `results/*.json`. |
| `outputs.py` | Regenerates the manuscript tables (`tables/*.tex`) and figures (`figures/*.pdf`) from the cached results. |

Generated artifacts are written to `results/`, `tables/`, and `figures/`. The
optional testbed map (`F0`) reads a country-outline GeoJSON from `data/`.

## Requirements

- Python 3.11 or later
- `gurobipy==13.0.2` with a valid Gurobi license (academic licenses are free)
- `numpy>=1.26`
- `matplotlib>=3.8`
- `cvxpy>=1.5` and `clarabel>=0.9` (only for the independent audit)

```
pip install "gurobipy==13.0.2" "numpy>=1.26" "matplotlib>=3.8" "cvxpy>=1.5" "clarabel>=0.9"
```

## Reproducing the results

Regenerate every cached result, then rebuild the tables and figures:

```
python experiments.py     # solves all experiments (licensed Gurobi required)
python outputs.py          # writes tables/*.tex and figures/*.pdf
```

A full solve of the six configurations and all sweeps takes a few minutes per
configuration on a single solver thread. The uncertainty study runs 100 Monte
Carlo draws and dominates the runtime.

If the cached `results/*.json` are present, `python outputs.py` regenerates all
tables and figures without Gurobi. Individual experiments can be run directly,
for example:

```python
from experiments import run_main, run_marketpower, run_uncertainty
run_main()
run_marketpower()
run_uncertainty("virum", seeds=100)
```

Each module can also be executed on its own for a quick check:

```
python config.py           # list the configurations
python data.py             # annual profile summary per configuration
python model.py            # waterfall and planner for the primary configuration
```

## Configurations

Four configurations are studied in depth (Virum, DK; Limerick, IE; Vasteras, SE;
Graz-Berlin, DE) and two are illustrative (Turkey, Romania). Each is sized to the
positive-energy threshold. Solver settings are fixed (single thread, seed 42,
optimality gap 1e-4) and every profile is deterministic given the seed, so the
reported numbers reproduce exactly.

## Data availability

All inputs are generated by `data.py` from the documented calibration constants,
and no third-party datasets are redistributed. `PROVENANCE.md` documents the
source organization, basis year, and transformation for each calibrated value. The archived release is deposited
with a persistent DOI and pins
the Gurobi and Clarabel versions above.
